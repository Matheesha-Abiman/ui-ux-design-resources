# 📱 UI/UX Responsive Design Constraints

In UI/UX tools like **Figma, Adobe XD, and Sketch**, constraints control how UI elements behave when the **screen size changes**. This helps designers create **responsive layouts**.

### Left Constraint
Element remains fixed to the **left side** of the container.

Example:
- Sidebar navigation
- Logo placement

### Right Constraint
Element stays attached to the **right side**.

Example:
- Profile icon
- Notification button

### Top Constraint
Element stays attached to the **top edge** of the screen.

Example:
- Navigation bar
- Header

### Bottom Constraint
Element stays attached to the **bottom edge**.

Example:
- Mobile bottom navigation
- Footer

### Left & Right Constraint
Element **stretches horizontally** when screen width changes.

Example:
- Search bars
- Input fields
- Cards

### Top & Bottom Constraint
Element **stretches vertically** when screen height changes.

Example:
- Background sections
- Panels

### Center Constraint
Element remains **centered** regardless of screen size.

Example:
- Modals
- Login forms
- Landing page titles

# 📐 Layout & Grid System

Most professional UI designs use a **grid system** to maintain alignment and consistency.

### 8px Spacing System
Designers commonly use **8px spacing increments**.

Common spacing values:
- 8px
- 16px
- 24px
- 32px
- 48px
- 64px

Benefits:
- Consistent spacing
- Clean visual structure
- Easier responsive scaling

# 📱 Responsive Breakpoints

Responsive design adapts UI to different screen sizes.

| Device | Width |
|------|------|
| Mobile | 320px – 480px |
| Tablet | 481px – 768px |
| Laptop | 769px – 1024px |
| Desktop | 1025px+ |

# 🎨 UI Design Principles

### Consistency
Use the same **colors, typography, icons, and spacing** across the product.

### Visual Hierarchy
Important elements should be **larger, brighter, or positioned prominently**.

### Simplicity
Avoid clutter. Keep interfaces **clean and minimal**.

### Feedback
Always give feedback for user actions.

Examples:
- Hover effects
- Loading indicators
- Success messages

# ♿ Accessibility Guidelines

Designs should be accessible to **all users**.

Best practices:
- Use **high color contrast**
- Minimum **16px text size**
- Clear button labels
- Keyboard navigation support
- Alt text for images
